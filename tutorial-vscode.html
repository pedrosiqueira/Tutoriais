VS Code Tutorial

Atalhos que mais utilizo

O mais importante de todos os atalhos é f1, pois ele abre um campo de busca em que você pode procurar por qualquer ação/comando do VSCode.

(Observação: atalhos para o teclado layout ABNT2).

  • ctrl+p Procura um arquivo no workspace atual.
  • ctrl+r Procura um arquivo recentemente aberto.
  • ctrl+shift+o Procura por um símbolo no arquivo atual.
  • ctrl+t Procura por um símbolo no workspace atual.
  • ctrl+mouse1 Vai para a declaração do símbolo clicado.
  • f2 Renomeia o símbolo atual.
  • ctrl+g Vai para uma linha específica.
  • alt+left Volta o cursor para onde estava anteriormente.
  • alt+right Avança o cursor para onde estava posteriormente.
  • ctrl+; Alterna comentário nas linhas selecionadas.
  • ctrl+. Correção de código.
  • ctrl+space Sugestão de código.
  • ctrl+k ctrl+i Exibe documentação.
  • ctrl+shift+space Exibe dicas de parâmetros.
  • ctrl+d Seleciona a palavra atual. Teclando múltiplas vezes, vai selecionando a mesma palavra no decorrer do arquivo.
  • alt+mouse1 Adiciona um cursor onde clicou. Esse atalho não é editável. Se não funcionar, acesse o menu "Selection → Switch to Ctrl+Click for Multi-Cursor".
  • ctrl+shift+mouse1 (linux) ou shift+alt+mouse1 (windows). Seleção retangular.
  • ctrl+shift+up (linux) ou ctrl+alt+up (windows). Adiciona um cursor na linha superior.
  • ctrl+shift+down (linux) ou ctrl+alt+down (windows). Adiciona um cursor na linha inferior.
  • ctrl+u Volta o cursor para a última posição.
  • alt+shift+left/right Expande/encolhe seleção atual.
  • ctrl+enter Cria uma linha embaixo e vai para ela.
  • ctrl+l Seleciona uma linha inteira.
  • ctrl+shift+e Abre a aba de arquivos.
  • ctrl+shift+g Abre a aba de versionamento de código.
  • ctrl+shift+x Abre a aba de extensões.
  • ctrl+b Alterna a visibilidade da aba lateral.
  • ctrl+h Abre o campo de substituição de palavras.
  • ctrl+alt+enter Substitui todas as palavras do campo de substituição.
  • ctrl+, Abre as configurações.
  • ctrl+j Alterna a visibilidade do painel.
  • ctrl+shift+´ (linux) ou ctrl+' (windows) Foca o painel no terminal (também alterna a visibilidade do terminal).
  • ctrl+k ctrl+h (linux) ou ctrl+shift+u (windows) Foca o painel na janela de saída (também alterna a visibilidade da janela de saída).
  • ctrl+shift+up (linux) ou ctrl+alt+pgup (windows) Sobe uma linha no terminal.
  • ctrl+shift+down (linux) ou ctrl+alt+pgdown (windows) Desce uma linha no terminal.
  • ctrl+pgup Volta pro terminal/arquivo anterior, dependendo de onde está focado.
  • ctrl+pgdown Vai pro próximo terminal/arquivo, dependendo de onde está focado.
  • ctrl+shift+[ Expande trecho atual.
  • ctrl+shift+´ Encolhe trecho atual.
  • ctrl+k ctrl+- Encolhe tudo menos trecho atual.
  • ctrl+k ctrl+0 Encolhe tudo.
  • ctrl+k ctrl+j Expande tudo.
  • ctrl+k ctrl+shift+n Foca no balão de notificações (aperte ESC para fechá-lo).

Atalhos comuns a outros editores de texto

  • ctrl+c Copia o que estiver selecionado. Se nada estiver selecionado, copia a linha atual.
  • ctrl+x Recorta o que estiver selecionado. Se nada estiver selecionado, recorta a linha atual.
  • ctrl+v Cola o que foi copiado ou recortado.
  • ctrl+backspace/delete Apaga palavra antes/depois do cursor.
  • home/end Vai para início/final da linha.
  • ctrl+home/end Vai para início/depois do documento.
  • shift+home/end Seleciona o que está antes/depois do cursor na mesma linha.
  • ctrl+shift+home/end Seleciona o que está antes/depois do cursor no documento.
  • setas Navega pelos caracteres.
  • ctrl+setas Navega pelas palavras.
  • shift+setas Seleciona por caractere.
  • ctrl+shift+setas Seleciona por palavras.
  • ctrl+a Seleciona tudo.
  • ctrl+s Salva arquivo atualmente aberto.
  • ctrl+tab Troca para próxima aba.
  • ctrl+shift+tab Troca para aba anterior.
  • alt+n Troca para nª aba, sendo n um número.
  • f11 Alterna tela cheia.

Atalhos personalizados

Muitas ações não vêm com um atalho associado, ou pode ser que você queira mudar o atalho de determinada ação. Para explorar e editar todos os atalhos, tecle f1, digite "shortcut", e escolha a opção "Open Keyboard Shortcuts". Atalhos personalizados são salvos num arquivo JSON citado na seção Configurando alguns atalhos.

Dicas

  • Ao criar um arquivo (atalho ctrl+n), você pode passar o caminho do arquivo, que, se não existir, o vscode já cria pra você.
  • "Leitura Obrigatória" para quem trabalha com HTML: Plugin Emmet já instalado no VSCode para edição de HTML.

Configurando o VS Code

Ao abrir o menu de configurações (ctrl+,), você pode procurar por qualquer configuração digitando no campo de busca. Por exemplo, para ativar a quebra virtual automática de linhas, ao digitar "wrap", deve aparecer uma configuração "Editor: Word Wrap", onde você pode alterar para "on".

Configurações personalizadas são salvas num arquivo JSON, que no Linux fica localizado em ~/.config/Code/User/settings.json. Para abrir o arquivo de configurações, você também pode teclar f1, digitar "sett js", e selecionar "Preferences: Open User Settings (JSON)". Eu, particularmente, gosto dessas configurações personalizadas, use-as se lhe convir:

    {
        "editor.wordWrap": "on", // habilita quebra virtual de linhas
        "editor.mouseWheelZoom": true, // altera o zoom com a rodinha do mouse
        "editor.unicodeHighlight.nonBasicASCII": false, // não destaca palavras com acento
        "workbench.editor.empty.hint": "hidden", // desativa mensagem chata quando cria novo arquivo
        "security.workspace.trust.enabled": "false", // confia em qualquer diretório por padrão
        "html.format.wrapLineLength": 0 // não quebra linhas do html automaticamente
    }

Configurando alguns atalhos

Abra o arquivo de atalhos (configuração "Open Keyboard Shortcuts (JSON)" ou caminho ~/.config/Code/User/keybindings.json se estiver no linux) para configurar os atalhos. Coloque os seguintes atalhos que lhe convir:

// Place your key bindings in this file to override the defaultsauto[]
[
    { // indenta código
        "key": "shift+alt+f",
        "command": "editor.action.formatDocument",
        "when": "editorHasDocumentFormattingProvider && editorTextFocus && !editorReadonly && !inCompositeEditor"
    },
    { // aumenta o tamanho da fonte apenas do código // ctrl+oem_plus no windows
        "key": "ctrl+=",
        "command": "editor.action.fontZoomIn"
    },
    { // diminui o tamanho da fonte apenas do código // ctrl+oem_minus no windows
        "key": "ctrl+-",
        "command": "editor.action.fontZoomOut"
    },
    { // exibe o arquivo ativo na aba explorer
        "key": "ctrl+1",
        "command": "workbench.files.action.showActiveFileInExplorer"
    },
    { // foca na aba explorer quando nenhum arquivo estiver ativo
        "key": "ctrl+1",
        "command": "workbench.view.explorer",
        "when": "!editorTextFocus"
    },
    { // envolve o texto selecionado ou a linha atual em uma tag html
        "key": "ctrl+alt+1",
        "command": "editor.emmet.action.wrapWithAbbreviation"
    },
    { // tem que ter um jeito mais fácil de deletar linha
        "key": "ctrl+alt+x",
        "command": "editor.action.deleteLines",
        "when": "textInputFocus && !editorReadonly"
    },
    { // voltar o cursor para onde estava
        "key": "alt+left",
        "command": "workbench.action.navigateBack"
    },
    { // avançar o cursor para onde estava
        "key": "alt+right",
        "command": "workbench.action.navigateForward",
        "when": "canNavigateForward"
    },
    { // copia o caminho do arquivo atual
        "key": "ctrl+alt+c",
        "command": "copyFilePath",
        "when": "editorFocus"
    },
    { // alterna comentário na(s) linha(s) selecionada(s)
        "key": "ctrl+;",
        "command": "editor.action.commentLine",
        "when": "editorTextFocus && !editorReadonly"
    },
    { // alterna a visibilidade da barra de atividades (barra lateral) // "ctrl+abnt_c1" no windows
        "key": "ctrl+/",
        "command": "workbench.action.toggleActivityBarVisibility"
    },
    { // sobrescreve comando do windows. mas no linux já é esse
        "key": "ctrl+shift+up",
        "command": "editor.action.insertCursorAbove",
        "when": "editorTextFocus"
    },
    { // sobrescreve comando do windows. mas no linux já é esse
        "key": "ctrl+shift+down",
        "command": "editor.action.insertCursorBelow",
        "when": "editorTextFocus"
    },
    { // encerra o terminal atual
        "key": "ctrl+k delete",
        "command": "workbench.action.terminal.kill"
    },
    { // alterna maximização do painel
        "key": "ctrl+f10",
        "command": "workbench.action.toggleMaximizedPanel"
    },
    { // remap toggle bold on extension yzhang.markdown-all-in-one
        "key": "ctrl+k b",
        "command": "markdown.extension.editing.toggleBold",
        "when": "editorTextFocus && !editorReadonly && editorLangId =~ /^markdown$|^rmd$|^quarto$/"
    },
    { // remap toggle bold on extension yzhang.markdown-all-in-one
        "key": "ctrl+b",
        "command": "-markdown.extension.editing.toggleBold",
        "when": "editorTextFocus && !editorReadonly && editorLangId =~ /^markdown$|^rmd$|^quarto$/"
    },
    { // remap show preview on extension yzhang.markdown-all-in-one
        "key": "ctrl+alt+v",
        "command": "markdown.showPreview",
        "when": "!notebookEditorFocused && editorLangId == 'markdown'"
    },
    { // remap show preview on extension yzhang.markdown-all-in-one
        "key": "ctrl+shift+v",
        "command": "-markdown.showPreview",
        "when": "!notebookEditorFocused && editorLangId == 'markdown'"
    }
]

Extensões

Para instalar um extensão, basta abrir a aba de extensões (ctrl+shift+x) e digitar o nome da extensão. Vamos exemplificar instalando a extensão rioj7.select-by. Essa extensão habilita atalhos para buscar textos usando regex. Depois abra o arquivo de configurações (~/.config/Code/User/settings.json ou ação "Open Settings (JSON)") e adicione o seguinte código:

    , "selectby.regexes": { // https://stackoverflow.com/a/63061477/4072641
        "goToNextTag": {
            "moveby": "<.*?>|.(?=<.*?>)"
        },
        "goToPreviousTag": {
            "moveby": "<.*?>|.(?=<.*?>)"
        }
    },

Esse código adiciona duas configurações, goToNextTag e goTopreviousTag, que movem o cursor para o início/final da próxima/anterior tag. Agora, vamos mapear os atalhos ctrl+alt+. e ctrl+alt+, para essas duas novas configurações, respectivamente. Adicione o seguinte código no arquivo de atalhos:

    , { // https://stackoverflow.com/a/63061477/4072641
        "key": "ctrl+alt+.", // go to next, whatever keybinding you want
        "command": "moveby.regex",
        "args": [
            "goToNextTag",
            "moveby",
            "next",
            "end",
            "wrap"
        ],
        "when": "editorTextFocus && editorLangId == html" // if html only
    },
    { // https://stackoverflow.com/a/63061477/4072641
        "key": "ctrl+alt+,", // go to previous
        "command": "moveby.regex",
        "args": [
            "goToPreviousTag",
            "moveby",
            "prev",
            "end",
            "wrap"
        ],
        "when": "editorTextFocus && editorLangId == html" // if html only
    }

Experimente criar ou abrir um arquivo HTML pelo VSCode e navegar rapidamente pelas tags usando os atalhos configurados.
